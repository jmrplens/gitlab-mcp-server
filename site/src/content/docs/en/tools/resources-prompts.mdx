Beyond tools, GitLab MCP Server exposes **resources** and **prompts** — two additional MCP primitives that provide context and reusable templates to AI assistants.

## Resources

MCP resources provide **read-only context data** that clients can request at any time without invoking a tool. Resources are useful for providing background information that helps the LLM make better decisions.

The server exposes **24 resources** across several categories:

### User Information

| Resource | Description |
|----------|-------------|
| `gitlab://user/current` | Current authenticated user profile |
| `gitlab://user/status` | Current user's status message |
| `gitlab://user/preferences` | User preferences and settings |

### Project Information

| Resource | Description |
|----------|-------------|
| `gitlab://project/{id}` | Project details (name, description, visibility, default branch) |
| `gitlab://project/{id}/languages` | Programming language breakdown |
| `gitlab://project/{id}/members` | Project team members |
| `gitlab://project/{id}/labels` | Available project labels |
| `gitlab://project/{id}/milestones` | Project milestones |
| `gitlab://project/{id}/branches` | Branch list with protection status |
| `gitlab://project/{id}/merge_requests` | Open merge requests summary |
| `gitlab://project/{id}/pipelines` | Recent pipeline status |
| `gitlab://project/{id}/variables` | CI/CD variable names (values hidden) |

### Server & Configuration

| Resource | Description |
|----------|-------------|
| `gitlab://server/version` | GitLab server version and revision |
| `gitlab://server/features` | Available GitLab features |
| `gitlab://server/metadata` | Server metadata (edition, enterprise flag) |
| `gitlab://config/current` | Current MCP server configuration (secrets redacted) |

### Tool Discovery

| Resource | Description |
|----------|-------------|
| `gitlab://tools/list` | All registered tools with descriptions |
| `gitlab://tools/categories` | Tools organized by domain category |
| `gitlab://tools/meta` | Meta-tool action reference |

:::note
Resource URIs with `{id}` are **resource templates**. The client substitutes the actual project ID or path when requesting the resource.
:::

### Accessing Resources

MCP clients can request resources at any time using the `resources/read` method:

```json
{
  "method": "resources/read",
  "params": {
    "uri": "gitlab://user/current"
  }
}
```

The server returns the resource content as structured data.

### Using Prompts

Prompts are requested via the `prompts/get` MCP method:

```json
{
  "method": "prompts/get",
  "params": {
    "name": "review-merge-request",
    "arguments": {
      "project": "my-group/my-project",
      "mr_iid": "42"
    }
  }
}
```

The server returns a structured prompt with context-aware content that the LLM can use to guide its workflow.

## Configuration

| Variable | Default | Description |
|----------|---------|-------------|
| `META_TOOLS` | `true` | Resources and prompts are registered in both modes |

:::tip
Resources are particularly useful for providing project context at the start of a conversation. By reading the project resource, the LLM gains immediate understanding of the repository's purpose, team, and current state.
:::
