Más allá de las herramientas, GitLab MCP Server expone **recursos** y **prompts** — dos primitivas MCP adicionales que proporcionan contexto y plantillas reutilizables a los asistentes de IA.

## Recursos

Los recursos MCP proporcionan **datos de contexto de solo lectura** que los clientes pueden solicitar en cualquier momento sin invocar una herramienta. Los recursos son útiles para proporcionar información de fondo que ayuda al LLM a tomar mejores decisiones.

El servidor expone **24 recursos** en varias categorías:

### Información de usuario

| Recurso | Descripción |
|----------|-------------|
| `gitlab://user/current` | Perfil del usuario autenticado actual |
| `gitlab://user/status` | Mensaje de estado del usuario actual |
| `gitlab://user/preferences` | Preferencias y configuraciones del usuario |

### Información de proyecto

| Recurso | Descripción |
|----------|-------------|
| `gitlab://project/{id}` | Detalles del proyecto (nombre, descripción, visibilidad, rama predeterminada) |
| `gitlab://project/{id}/languages` | Desglose de lenguajes de programación |
| `gitlab://project/{id}/members` | Miembros del equipo del proyecto |
| `gitlab://project/{id}/labels` | Etiquetas disponibles del proyecto |
| `gitlab://project/{id}/milestones` | Milestones del proyecto |
| `gitlab://project/{id}/branches` | Lista de ramas con estado de protección |
| `gitlab://project/{id}/merge_requests` | Resumen de merge requests abiertas |
| `gitlab://project/{id}/pipelines` | Estado de pipelines recientes |
| `gitlab://project/{id}/variables` | Nombres de variables CI/CD (valores ocultos) |

### Servidor y configuración

| Recurso | Descripción |
|----------|-------------|
| `gitlab://server/version` | Versión y revisión del servidor GitLab |
| `gitlab://server/features` | Funciones disponibles de GitLab |
| `gitlab://server/metadata` | Metadatos del servidor (edición, flag enterprise) |
| `gitlab://config/current` | Configuración actual del servidor MCP (secretos redactados) |

### Descubrimiento de herramientas

| Recurso | Descripción |
|----------|-------------|
| `gitlab://tools/list` | Todas las herramientas registradas con descripciones |
| `gitlab://tools/categories` | Herramientas organizadas por categoría de dominio |
| `gitlab://tools/meta` | Referencia de acciones de meta-herramientas |

:::note
Las URIs de recursos con `{id}` son **plantillas de recursos**. El cliente sustituye el ID o ruta del proyecto real al solicitar el recurso.
:::

### Acceso a recursos

Los clientes MCP pueden solicitar recursos en cualquier momento usando el método `resources/read`:

```json
{
  "method": "resources/read",
  "params": {
    "uri": "gitlab://user/current"
  }
}
```

El servidor devuelve el contenido del recurso como datos estructurados.

## Prompts

Los prompts MCP son **plantillas reutilizables** que guían a los asistentes de IA a través de flujos de trabajo comunes. El LLM puede usar estas plantillas para estructurar sus respuestas y asegurar salidas consistentes y de alta calidad.

El servidor proporciona **38 plantillas de prompts** organizadas en categorías:

### Prompts principales (12)

Prompts de propósito general para flujos de trabajo cotidianos de GitLab.

| Prompt | Descripción |
|--------|-------------|
| `create-issue` | Creación guiada de issues con etiquetas y asignaciones correctas |
| `review-merge-request` | Checklist estructurada de revisión de MRs |
| `create-release` | Creación de releases con generación de changelog |
| `triage-issues` | Flujo de trabajo de triaje y priorización de issues |
| `pipeline-troubleshoot` | Guía paso a paso para depuración de pipelines |
| `branch-strategy` | Guía de nomenclatura y gestión de ramas |
| `code-review-checklist` | Plantilla completa de revisión de código |
| `sprint-planning` | Asistente de planificación de sprints |
| `incident-response` | Flujo de trabajo de respuesta a incidentes |
| `migration-guide` | Planificación de migración de proyectos |
| `security-audit` | Checklist de auditoría de seguridad |
| `onboarding-developer` | Guía de incorporación de nuevos desarrolladores |

### Uso de prompts

Los prompts se solicitan mediante el método MCP `prompts/get`:

```json
{
  "method": "prompts/get",
  "params": {
    "name": "review-merge-request",
    "arguments": {
      "project": "my-group/my-project",
      "mr_iid": "42"
    }
  }
}
```

El servidor devuelve un prompt estructurado con contenido consciente del contexto que el LLM puede usar para guiar su flujo de trabajo.

## Configuración

| Variable | Predeterminado | Descripción |
|----------|---------|-------------|
| `META_TOOLS` | `true` | Los recursos y prompts se registran en ambos modos |

:::tip
Los recursos son particularmente útiles para proporcionar contexto del proyecto al inicio de una conversación. Al leer el recurso del proyecto, el LLM obtiene comprensión inmediata del propósito del repositorio, el equipo y el estado actual.
:::
