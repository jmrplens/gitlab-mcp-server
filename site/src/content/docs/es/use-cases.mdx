## Flujos de trabajo

<CardGrid>
	<Card title="🔀 Revisión de código" icon="pencil">
		Revisa merge requests, analiza cambios, comprueba problemas de seguridad y deja comentarios — todo mediante lenguaje natural.

    	**Herramientas clave:** `gitlab_merge_request`, `gitlab_mr_changes`, `gitlab_mr_discussion`

    	**Prueba:** _"Resume los cambios del MR !42 y comprueba problemas de seguridad"_
    </Card>

    <Card title="🔄 Automatización CI/CD" icon="setting">
    	Monitoriza pipelines, depura fallos, gestiona variables CI y revisa programaciones de pipelines sin salir de tu editor.

    	**Herramientas clave:** `gitlab_pipeline`, `gitlab_job`, `gitlab_ci_variable`

    	**Prueba:** _"¿Por qué falló el último pipeline en la rama feature/auth?"_
    </Card>

    <Card title="📋 Gestión de issues" icon="list-format">
    	Crea, actualiza y haz seguimiento de issues. Gestiona etiquetas, hitos y asignaciones mediante conversación.

    	**Herramientas clave:** `gitlab_issue`, `gitlab_label`, `gitlab_milestone`

    	**Prueba:** _"Crea un bug report titulado 'Arreglar página de login' con etiqueta 'bug' y asignar a @alice"_
    </Card>

    <Card title="📦 Gestión de releases" icon="rocket">
    	Crea releases, genera changelogs, gestiona tags y sube assets de release.

    	**Herramientas clave:** `gitlab_release`, `gitlab_tag`, `gitlab_release_link`

    	**Prueba:** _"Genera notas de release comparando v1.0 con v2.0"_
    </Card>

    <Card title="👥 Gestión de equipo" icon="seti:ignored-vscode">
    	Gestiona miembros de proyecto, membresías de grupo y niveles de acceso.

    	**Herramientas clave:** `gitlab_user`, `gitlab_group`, `gitlab_member`

    	**Prueba:** _"Añade a @bob como developer en el proyecto my-app"_
    </Card>

    <Card title="🔍 Búsqueda de código" icon="magnifier">
    	Busca en código, issues, merge requests y wikis. Explora el árbol del repositorio y compara ramas.

    	**Herramientas clave:** `gitlab_search`, `gitlab_repository`, `gitlab_file`

    	**Prueba:** _"Busca todos los comentarios TODO en el proyecto my-app"_
    </Card>

    <Card title="📊 Informes y analítica" icon="open-book">

    	**Prompts clave:** standup, risk-assessment, workload, milestone-report

    	**Prueba:** _"Revisa el MR !42 en busca de vulnerabilidades de seguridad"_
    </Card>

## Ejemplos de flujos de trabajo

### Flujo de revisión de código

1. _"Muestra los MRs abiertos en my-app que necesitan revisión"_ — lista MRs esperando revisión
2. _"Resume los cambios del MR !42"_ — obtiene un resumen del diff
3. _"Comprueba el MR !42 por problemas de seguridad"_ — ejecuta análisis de seguridad
4. _"Deja un comentario en el MR !42 sugiriendo añadir validación de entrada"_ — añade feedback de revisión
5. _"Aprueba el MR !42"_ — aprueba el merge request

### Flujo de depuración CI/CD

1. _"¿Cuál es el estado del pipeline de my-app en main?"_ — comprueba el último pipeline
2. _"Muestra los jobs fallidos del pipeline #8847"_ — identifica las etapas con fallos
3. _"Dame los logs del job unit-tests del pipeline #8847"_ — obtiene detalles del error
4. _"¿Qué variables CI están configuradas en el proyecto my-app?"_ — comprueba la configuración
5. _"Reintenta los jobs fallidos del pipeline #8847"_ — re-ejecuta los fallos

### Flujo de planificación de sprint

1. _"Muestra el hito v3.0 del proyecto my-app"_ — revisa el alcance del hito
2. _"Lista los issues abiertos del hito v3.0"_ — ve el trabajo pendiente
3. _"Crea un issue para 'Implementar rate limiting' en el hito v3.0"_ — añade tareas faltantes
4. _"Asigna el issue #234 a @alice"_ — distribuye el trabajo
5. _"Genera un informe de progreso del hito v3.0"_ — crea resumen de estado
